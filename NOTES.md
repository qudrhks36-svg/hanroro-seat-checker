# 한로로 콘서트 좌석 알리미 (hanroro-seat-checker)

롯데콘서트홀 "SERIES.L : 한로로" 공연(2026-08-06, 2026-08-07) 좌석이 취소로 풀리는 순간 텔레그램으로 알려주는 자동화. 다른 PC에서 이 저장소를 열었을 때 왜 이렇게 구성되어 있는지 바로 파악할 수 있도록 정리.

## 구조

- `scripts/check_seats.py` — 롯데콘서트홀 스케줄 API(`/product/ko/performance/{id}/schedule`)를 직접 호출해서 회차별 매진 여부 확인. Playwright/브라우저 불필요.
- `.github/workflows/check-seats.yml` — GitHub Actions 워크플로. **트리거는 `workflow_dispatch`만 있음.**
- `state.json` — 마지막으로 "좌석 없음" 알림을 보낸 시각을 기록. 워크플로가 매 실행마다 커밋해서 저장소에 영속화됨 (`chore: update seat check state [skip ci]` 커밋들).

## 스케줄링: 왜 GitHub 자체 cron이 없는가

**실제 스케줄러는 저장소 밖의 cron-job.org.** 2분 간격으로 GitHub Actions API(`workflow_dispatch`)를 호출해서 실행시킴.

- GitHub 자체 `schedule:` cron을 같이 켰다가, cron-job.org와 중복 실행되는 문제가 있어서 제거함 (2026-07-16).
- 과거에 cron-job.org 설정을 끄면서 GitHub 쪽 대체 스케줄을 안 넣어 **완전히 자동 실행이 멈췄던 사고가 있었음** — cron-job.org 대시보드에서 이 job이 살아있는지, 시간대가 KST로 맞는지 주기적으로 확인 필요.
- **둘 다 켜지 말 것.** 하나만 켜져 있어야 함 (지금은 cron-job.org 단독).

## 알림 로직 (scripts/check_seats.py)

- **9시~18시(KST) 안에서만 동작.** 이 시간 밖이면 조회조차 안 하고 조용히 종료 — cron-job.org가 오작동해서 시간대 밖에 호출해도 안전하도록 스크립트 자체에 방어 로직을 넣음.
- **좌석 없음**: `state.json`의 마지막 알림 시각 기준 **30분 경과했을 때만** 알림 (매 2분마다 스팸 보내지 않기 위함).
- **좌석 발생**: 스로틀 무시하고 **즉시** 알림 + 회차별 좌석 선택 URL 포함.
- **2026-08-08 00:00 KST 이후**(마지막 공연 다음날): 자동으로 종료 알림을 보내고 `disable_workflow()`로 워크플로 자체를 비활성화함 (영구 방치 방지, 비용 0원 보장).

## 결제 속도 관련 (2026-07-16 대화에서 정리됨)

- **네이버 로그인 미지원.** 롯데콘서트홀은 L.POINT 통합회원 아이디/비번으로만 로그인 가능. 네이버 계정으로 우회 불가.
- 알림에 오는 좌석 URL(`SeatPreviewProcess.aspx`)은 **로그인 없이도 좌석 배치도까지는 바로 보임.** 로그인이 필요한 건 실제 좌석 선택 후 결제 단계.
- 결제를 빠르게 하려면 **알림 URL을 열 브라우저에 미리 로그인해두고 세션을 유지**하는 게 핵심. 네이버 앱 인앱브라우저 / 텔레그램 인앱브라우저 / 시스템 기본 브라우저는 쿠키가 서로 공유되지 않으므로, 텔레그램 링크를 길게 눌러 "브라우저에서 열기"로 **로그인해둔 바로 그 브라우저**로 열어야 함.
- **아이폰 사파리 로그인 시 흰 화면 멈춤 버그가 공식 FAQ에 등록되어 있음.** 공식 해결법이 사파리 방문기록/쿠키 전체 삭제라서, 결제 직전에 이 버그가 터지면 로그인 세션도 같이 날아가는 딜레마가 있음. 미리 세션을 살려두면(로그인 과정을 다시 안 타면) 이 버그를 피할 가능성이 높음. 리스크가 걱정되면 사파리 대신 아이폰 Chrome 사용 고려.

## 트러블슈팅

- 텔레그램 알림이 안 온다 → `gh run list --workflow=check-seats.yml` 로 최근 실행 확인. 실행 자체가 없으면 cron-job.org 쪽 문제, 실행은 있는데 메시지가 없으면 30분 스로틀 또는 9-18시 밖일 가능성.
- 워크플로가 `disabled` 상태다 → CUTOFF(2026-08-08) 지났거나, 수동으로 비활성화된 것. `gh api repos/qudrhks36-svg/hanroro-seat-checker/actions/workflows` 로 상태 확인.
